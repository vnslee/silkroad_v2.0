# 보고서 생성 엔진 명세서 (JSON 생성)

> **책임 범위:** 데이터 조사·계산·플래그·**데이터 성격**까지. JSON을 출력한다.
> **책임 밖:** 차트 유형·배지 색상·레이아웃 → 렌더링 엔진 명세서 참조.
> 경계 원칙: 생성은 "이 값이 무엇이고 어떤 성격인가"까지만. "어떻게 보여줄지"는 렌더링이 결정.

---

## 0. 공통 원칙

### 데이터 원천 플래그 (항목 단위, JSON에 포함)
- `[EXT]` 외부조사 — 웹·기관 자료로 조사한 외부 사실 (출처·Tier 필수, 검증 가능)
- `[INT]` 내부자료 — 캐피탈사 제공값 (별도 파일/스냅샷에서 읽어옴, 외부조사 금지)
- `[CALC]` 계산값 — 산식으로 도출 (결정론적·재현 가능)
- `[AI]` AI 인사이트 — 교차해석·추론 (검증 필요, **AI가 만든 것만**)
- `[NEWS]` 외부이슈 — 로이터 등 공신력 언론 (시점·출처 필수)
> ※ 조사 대상은 `[EXT]`·`[NEWS]`뿐. `[AI]`는 조사 아님(다른 탭 결과 해석). `[CALC]`는 연산, `[INT]`는 읽기.

### 데이터 성격 (항목 단위, JSON에 포함 — 렌더링이 차트 선택에 사용)
- `timeseries` 시계열 — 연도별 추세 (GDP·CAGR·EV보급·잔가·TCO누적)
- `composition` 구성비 — 비율 합산 (상품믹스·구매패턴·세그먼트)
- `ranking` 순위 — 비교·줄세우기 (브랜드·OEM·금융사·국가점수)
- `score_multiaxis` 다축점수 — 유사도 축별 (탭1-1 4축 / 탭2-2 국가×축)
- `single_value` 단일수치 — KPI (TCO·종합점수)
- `qualitative` 정성 — 서술 (시장특징·화이트스페이스)
- `status_matrix` 상태표 — 통과/탈락 (킬스위치)
> 생성 엔진은 **성격까지만** JSON에 담는다. 차트 유형(라인/도넛/바…) 결정은 렌더링 몫.

### 근거 표기 의무
- `[EXT]`·`[NEWS]`: 값 + 출처(기관·문서·날짜) + Tier. `[INT]`: 스냅샷 ID. `[CALC]`: 산식·입력 추적. `[AI]`: 근거 탭/수치 참조.
- 못 찾으면 "조사 필요(실사)" — 지어내지 말 것.

### 가중치 (유형2 전용)
- `config.json`의 `values`에서 읽어옴(EXT/CALC 아님, 정책 입력값). 유형1은 가중치 없음.

### 통화 처리 (국가 비교 핵심)
- **기준통화 = KRW(원화).** 본사 의사결정용. 모든 금액은 최종적으로 KRW로 환산해 비교.
- **표시 규칙:** `KRW 환산값 + (현지통화 원값)` 병기. 예: `₩3,200만 (PLN 92,000)`.
- **원천별 통화:**
  - 현지통화 조사값 `[EXT]` (평균신차가격·시장규모·경쟁사금리 등) → 현지통화로 조사 후 KRW 환산
  - 구독료·솔루션 계약 `[INT]` → **EUR 계약 원본 유지** 후 KRW 환산 (환산 전 EUR 원값도 보존)
  - B국 구축실적 `[INT]` → 계약통화(GBP/EUR 등) 원본 유지 후 KRW 환산
- **환율은 관리자 입력(config) → 스냅샷 복사 고정.** 자동 조사 ❌. 관리자가 config에 입력한 환율을 스냅샷 생성 시 복사해 박는다 → 같은 스냅샷이면 같은 환율로 같은 결과(재현·감사). 생성 시점 실시간 환율 사용 금지(재현성 깨짐).
- **환율 메타:** 스냅샷에 `{ "base":"KRW", "rates": {"PLN":360.5, "EUR":1450.2, "GBP":1680.0, ...}, "fx_date":"2026-03-31" }` 형태로 기록(config에서 복사). fx_date도 보고서에 표기.
- **유형2 다국 비교:** 권역 내 모든 국가 금액을 동일 스냅샷 환율로 KRW 환산해야 비교 성립(국가마다 다른 환율 적용 금지).

# 유형 1 — 단일국가 비용·TCO

> 전제: 비즈니스 킬스위치 통과 후 진입. 킬스위치 탭 없음.
> 생성 순서: ①데이터(EXT·NEWS 조사 + INT 읽기) → ②계산(CALC) → ③해석(AI) → ④요약·카드.

## [탭 1-1] 유사도 점수 (A vs B)
> **채점 원칙:** 텍스트 일치(글자가 같나)가 아니라 **속성(디멘전) 단위로 분해해 기능적 유사도를 점수화**한다. 예) 영국 "Consumer Credit Authorisation" vs 폴란드 "KNF 등록" — 글자는 달라도 둘 다 비은행 여신 라이선스이므로, 취득난이도·외국인가능·처리기간 등 속성으로 비교하면 높은 유사도가 나온다. 각 디멘전을 A·B 각각 점수화 → 디멘전별 격차 → 항목 유사도 → 축 유사도 → 종합 유사도(100점 만점).

**[유사도 채점표 — 비교 디멘전]**

| 항목 | 비교 디멘전 (이 기준으로 A vs B 점수화) | 원천 |
|---|---|---|
| 솔루션 유형 + 코어 벤더 락인 | 배포형태(패키지/SI/SaaS) · 커스터마이징 자유도 · 벤더 종속도 · 멀티테넌시 여부 | `[EXT]` |
| 디지털 채널 성숙도 | 온라인 신청 연동 · 비대면 계약 가능 · API 개방도 · 페이퍼리스 수준 | `[EXT]` |
| 상품판매 현황(리스/렌탈/플릿) | 3종(리스·렌탈·플릿) 취급 일치도 · 각 상품 비중 유사도 | `[EXT]` |
| 라이선스 종류 + 소관 규제기관 | 취득방식(등록제 vs 인가제) · 외국인 취득 가능 · 처리기간(개월) · 최저자본금 수준 · 감독 강도 | `[EXT]` |
| 데이터현지화 + 개인정보보호법 | 현지 저장 강제 여부 · 국외이전 허용도 · 동의·보관 규제 강도 · GDPR 동등성 | `[EXT]` |
| 차량회수 절차 + 법적회수 기간 | 사법절차 필요 여부 · 회수 소요기간(일) · 자력구제 허용 · 회수율 | `[EXT]` |

> **채점 가이드:** 각 디멘전을 A·B에 대해 동일 척도(예: 1~5)로 평가 → 격차가 작을수록 고점. 디멘전 평균이 항목 유사도, 항목 평균이 축 유사도, 축 가중평균이 종합 유사도. 종합 점수가 탭1-2 분기(70/50)와 탭1-3 승수(%)의 입력값.
> ※ 축 가중평균의 계수는 유사도 산출의 **내부 계산값**(전략 정책값 아님) → 본문 산식에 고정. 화면/설정 파일 관리 대상 아님. (유형1은 금액 계산 보고서라 가중치 설정 없음)

> **★ 데이터-산식 분리 원칙 (탭1-1 ↔ 탭2-2 공통):**
> - **데이터는 한 벌만 관리.** 위 채점표의 디멘전 단위로 나라별 값을 1회 조사해 공유. 유형1·유형2가 같은 데이터를 참조한다. (중복 조사 금지)
> - **산식은 탭별로 다름.** 같은 데이터를 탭1-1은 정밀 채점, 탭2-2는 밴드 채점으로 처리한다.
>   - **탭1-1 (정밀):** 각 디멘전 1~5점 → 가중평균 → **0~100점 절대 유사도**. 단일국 심층, 70/50 분기·승수의 입력.
>   - **탭2-2 (밴드):** 같은 디멘전을 상/중/하 3밴드로 거칠게 → 합산 → **0~100점으로 환산하되 10점 구간(80점대/70점대)으로만 표기.** 다국 스크리닝·순위용.
> - **척도 통일:** 두 탭 모두 **100점 만점 공유**. 따라서 유형2 스크리닝 점수와 유형1 실사 점수를 직접 비교 가능(예: 스크리닝 82 → 실사 78, 갭 4점=규제 특화분). 깔때기 검증이 숫자로 연결됨.
> - **착시 방지:** 탭2-2는 밴드 기반이므로 소수점·1점 단위 비교 금지. 반드시 10점 구간으로 표기해 과잉정밀 착시를 막는다.

**[채점 대상 항목]** (전부 `[EXT]` 외부조사 → 산출되는 유사도 점수는 `[CALC]`)
- 솔루션 유형(패키지/SI/SaaS) + 코어 벤더 락인 `[EXT]`
- 디지털 채널 성숙도 `[EXT]`
- 상품판매 현황(리스/렌탈/플릿) `[EXT]`
- 라이선스 종류 + 소관 규제기관 `[EXT]`
- 데이터현지화 + 개인정보보호법 `[EXT]`
- 차량회수 절차 + 법적회수 기간 `[EXT]`
- → 종합 유사도 점수 `[CALC]`

**산출:** 종합 유사도 점수 `[CALC]` (성격: `single_value`) + 축별 유사도 `[CALC]` (성격: `score_multiaxis`)
> 종합 점수 → 탭1-2 분기(70/50)·탭1-3 승수(%)의 입력.

## [탭 1-2] 시스템 결정 트리
> B시스템 확산 / 외부솔루션 / 본사구축 분기. **분기 조건이 핵심 — 데이터 추출 시 필수.**

**특화요건이란:** B국 시스템을 A국에 그대로 가져다 쓸 수 있는지 가르는 체크포인트. A국에만 있는 상품·규제 요구가 B시스템에 없으면 추가 개발이 필요하며, 이 양·복잡도가 ① 유사도 점수를 깎고 ② 탭1-3의 특화개발비로 연결된다. (유사도 승수를 100%가 아닌 50~60%로 잡는 근거가 바로 이 특화개발 필요성)

**[입력 단계]**

| 단계 | 입력 항목 | 원천 |
|---|---|---|
| ① B시스템 식별 | 주요 솔루션사 · 솔루션 유형 · 코어 벤더 락인 | `[EXT]` |
| ② 특화요건 확인 (A국 고유 요구) | 상품판매 현황 / 개인정보보호법 · 데이터현지화 · 의무보험 · 신용생명보험 · 보험끼워팔기 · AI 신용평가 규제 | `[EXT]` |

**[분기 로직]** — 권역 내 기본국(B) 존재 전제. ②의 특화요건을 A국 정보와 대조해 유사도 점수 산출 후 분기.

| 조건 | 판정 | 처리 |
|---|---|---|
| 유사도 **70점 이상** | B 시스템 확산 | B시스템 + A국 특화 추가개발분 비용 산정 → 탭1-3 |
| 유사도 **50점 미만** | 외부솔루션 | A국 오토파이낸스사 주 사용 솔루션 + 로컬 솔루션 **2~3종 추천**. 각 솔루션이 커버하는 기능·인프라·규제를 A국 요건과 비교, **기준점 이상 시 추천** |
| 그 외 (50~70 / 외부솔루션도 기준 미달) | 본사 자체구축 | 본사 시스템으로 구축. 기본 사용 비용·기간은 제공값 사용 (참고자료로 제시) |

> ※ 본사 자체구축 비용·기간은 어느 분기든 **참고용으로 병기**(사용자 원 요건: "본사 시스템 구축 시 대략 비용도 참고자료로 제공").

**산출:** 시스템 결정 결과 `[CALC]` (성격: `qualitative`+분기경로).

## [탭 1-3] 계약건수·구독료 + 10년 TCO

**[입력 항목]**
- 자동차 판매대수 (A·B) `[EXT]`
- 금융 침투율(신차) `[EXT]`
- 구매패턴(현금:할부:리스) `[EXT]`
- B국 구축 실적(구축비용·기간) `[INT]`
- 구독료 구간표(누적건수 구간별 단가) `[INT]`
- 본사 자체구축 기본(비용·기간) `[INT]`
- 운영비 통금액(10년) `[INT]`
- 우리사 예상 점유율 가정 `[INT]`

**[산식 1] 유사도% 매핑** (탭1-1 종합점수 → 승수)

| 종합 유사도 | 승수 % |
|---|---|
| 90~100 | 50% |
| 80~90 | 60% |
| 70~80 | 70% |
| (이하 정책표 연장) | … |

**[산식 2] A국 예상 계약건수**
```
A 예상건수 = A 판매대수 × A 금융침투율 × (할부+리스 비중) × 우리사 예상점유율[INT]
```
> B 건수도 동일식 또는 B 실적값[INT] 사용. (사용자 예시: 영국 80건 ← 이 식으로 환산)

**[산식 3] 구독료 — 전체 소급**
```
신규 누적건수 = 기존 솔루션 전체국 누적건수[INT] + A 예상건수
적용 단가   = 구독료 구간표[INT]에서 '신규 누적건수'가 속한 구간의 단가
연 구독료   = 신규 누적건수 전체 × 적용 단가   ← 기존 물량도 새 단가로 소급
```
> 예: 기존 80건(@1유로=80유로) + 신규 80건 = 160건 → 구간단가 0.9유로 → **160건 전체 × 0.9 = 144유로/년** (기존 80건도 0.9로 소급)

**[산식 4] 10년 TCO 빌드업**
```
구축비(일회성)   = B 구축비용[INT] × 유사도%(산식1)
특화개발비(일회성) = 탭1-2 특화요건 복잡도 기반 추가개발 견적
연 유지비(반복)   = 연 구독료(산식3) + 유지보수[INT]
구축기간        = B 구축기간[INT] × 유사도%(산식1)
10년 TCO = 구축비 + 특화개발비 + (연 유지비 × 10) + 운영비 통금액[INT]
```
**산출:** 예상 계약건수·연 구독료 `[CALC]`(`single_value`) / 10년 TCO `[CALC]`(`single_value`) / TCO 구성분해 `[CALC]`(`composition`) / 연차 누적비용 `[CALC]`(`timeseries`) / 구독료 구간 `[CALC]`(`timeseries`성)
> ※ 운영비는 본 시스템 산식과 분리된 내부 통금액. 출력 시 "TCO = 시스템(구축+특화+유지) + 운영(내부 통금액)" 구분 표기.
> ※ 본사 자체구축 비용·기간[INT]은 결정트리 분기와 무관하게 참고용 병기.

## [탭 1-4] 시장·경쟁 배경 (참고)
- 금융사 순위 + 캡티브 강도 `[EXT]` (성격: `ranking`)
- 경쟁사 금리범위 + 점유율 + 진출형태 `[EXT]` (성격: `ranking`/범위)
- OEM 순위 / 평균 신차가격 `[EXT]` (성격: `ranking` / `single_value`)

- EV 보급률 + EV·ICE 잔존가치 리스크 `[EXT]` (성격: `timeseries`)

## [탭 1-0] 요약 (Executive Summary)

- **A. 핵심 결론** (타 탭 인용): 종합 유사도(탭1-1)·시스템 결정(탭1-2)·10년 TCO·기간(탭1-3) — 원 플래그 유지(`[CALC]`)

# 유형 2 — 권역 퀵윈 순위표

> 생성 순서 동일: ①데이터 → ②계산 → ③해석 → ④요약·카드.

## [탭 2-0] 킬스위치 필터
- 외국인 지분 한도 / 외환·배당 송금 / 데이터 현지화 / 국가신용등급 `[EXT]`
**산출:** 국가×킬스위치 통과/탈락 `[CALC]` (성격: `status_matrix`). 탈락국은 이후 스코어링에서 제외.

## [탭 2-1] 비즈니스 매력도
> **규칙:** 탭1-1·2-2와 동일 — 데이터 한 벌 공유, **0~100점 척도**, 항목별 정규화 후 가중평균.
> 단 매력도는 '유사도'가 아니라 '시장 자체 매력'이므로 ① 방향성(역점수) ② 단위 정규화를 추가 처리.

| 항목 | 방향 | 비교 디멘전 | 원천 |
|---|---|---|---|
| GDP 성장률 | 高=好 | 실질성장률(%) | `[EXT]` |
| 자동차 판매대수 + 시장규모(CAGR) | 高=好 | 시장 절대규모 · 성장추세 | `[EXT]` |
| 금융 침투율 + 금융이용유형 | 高=好 | 침투율(%) · 할부·리스 비중(우리 타깃상품 적합도) | `[EXT]` |
| 경쟁강도(토요타·VW 진출 + 금융사순위·점유율) | 高=惡(역점수) | 캡티브 선점 여부 · 1위사 점유율 · 시장 집중도 | `[EXT]` |
| 계약방식(디지털 vs 페이퍼) | 高=好 | 디지털 성숙도(진출 용이성) | `[EXT]` |

**[산식 1] 항목 정규화 (0~100)**
```
정점수 항목(高=好): 점수 = (해당국 값 − 권역 최저) / (권역 최고 − 권역 최저) × 100
역점수 항목(高=惡): 점수 = 100 − 위 정규화값   ← 경쟁강도에 적용
정성 항목(계약방식 등): 디지털=高 / 혼합=中 / 페이퍼=低 → 100/50/0 매핑
```
> 정규화 기준(최저~최고)은 **권역 내 후보국 집합** 기준. 국가 추가/제외 시 재정규화.

**[산식 2] 매력도 종합 (0~100)**
```
매력도 = Σ(항목 정규화점수 × 항목 가중치)
```
> 가중치: `config.json`의 `values.biz_attractiveness`. 예: 성장성 중시면 GDP·CAGR↑, 수익성 중시면 침투율·이용유형↑.

**산출:** 국가별 매력도 `[CALC]` (성격: `ranking`) + 항목 기여분 `[CALC]` (성격: `composition`) + GDP·CAGR 추이 `[EXT]` (성격: `timeseries`)

## [탭 2-2] IT/진출속도 유사도 (B국 대비)
> **데이터:** 탭1-1 채점표의 디멘전을 그대로 참조(한 벌 공유, 재조사 없음).
> **산식:** 탭1-1과 달리 밴드(상/중/하) 채점 → 0~100점 환산 → **10점 구간으로만 표기.** 권역 내 다국 줄세우기용.
> B와 유사할수록 시스템 확산 빨라 퀵윈. IT가 비즈니스에 역제안하는 탭.

채점 대상 항목(= 탭1-1 항목 중 IT·진출속도 핵심, 전부 `[EXT]` / 산출 IT유사도 `[CALC]`):
- 솔루션 유형 + 디지털 채널 성숙도 `[EXT]`
- 라이선스 종류 + 데이터현지화 `[EXT]`
- 상품판매 현황 `[EXT]`
- 차량회수 절차 `[EXT]`
> 가중치: `config.json`의 `values.it_readiness`.

**[최종 퀵윈]**
```
퀵윈 점수 = 매력도(2-1) × w_biz + IT유사도(2-2) × w_it    [config: values.report_blend]
```
> 킬스위치 탈락국 제외. 10점 구간 표기(과잉정밀 금지).
**산출:** 국가×4축 밴드 `[CALC]`(`score_multiaxis`) / 최종 퀵윈 순위 `[CALC]`(`ranking`) / 매력도×IT유사도 좌표 `[CALC]`(2축)

**[국가 프로파일 — 상위 3개국]** (새 조사 없이 타 탭 종합)
- 순위·국가명 + 퀵윈 점수(구간) `[CALC]` / 매력도·IT유사도 `[CALC]`
- 시장 한 줄: 판매대수·침투율·EV보급 `[EXT]`
- 경쟁: 토요타·VW 진출 + 1위사 `[EXT]`
- 킬스위치 통과 상태 `[EXT]`
- 핵심 이슈 1건 `[NEWS]`
- AI 한 줄 코멘트 `[AI]`
> 4위 이하는 순위표에만. 카드는 상위 3개국 한정.

## 📋 보고서 생성 규칙

### 명명 규칙
```
RPT_{유형}_{대상코드}_{NNN}
유형: CTR(유형1·국가) / RGN(유형2·권역)
대상: 국가=ISO 3166-1 alpha-2 / 권역=사내 코드표(EU·NA·SA·APAC)
```

### 저장 경로
```
유형1 : storage/report/country/{ISO2}/RPT_CTR_{ISO2}_{NNN}.json
유형2 : storage/report/region/{RGN}/RPT_RGN_{RGN}_{NNN}.json
```
- 입력 스냅샷은 `storage/data/...`, 출력은 `storage/report/...`. 대상 폴더 없으면 생성.

### 채번 로직 ("새로 생성" 클릭)
```
1) 대상 폴더 스캔: storage/report/{country|region}/{코드}/
2) RPT_..._{NNN} 패턴에서 NNN 최대값 추출 (없으면 000)
3) 신규 번호 = 최대값 + 1, 3자리 zero-pad (001, 002 …)
```
> 일련번호는 **대상 코드별로 독립**(PL의 001과 GB의 001은 별개). 시점 구분은 내부 메타 `data_snapshot_id`로.

### 필수 메타데이터
```json
{
  "report_id": "RPT_CTR_PL_001",
  "report_type": "type1_country",
  "title": "폴란드 오토파이낸스 진출 비용·TCO 보고서",
  "target": { "country": "PL", "base_country": "GB" },
  "generated_at": "2026-06-21T14:30:00+09:00",
  "generated_by": "전략기획팀 / 요청자",
  "data_snapshot_id": "SNAP_PL_2026Q1",
  "config_version": "CFG_2026-06-15",
  "engine_version": "v1.0"
}
```
- 유형2 target: `{ "region": "EU", "evaluated_countries": ["PL","CZ","HU", ...] }`

### 스냅샷 원칙
- 국가 데이터(시장·규제·시스템·**뉴스 포함**)는 **한 번에 조사·갱신 = 스냅샷 1개**.
- 갱신 시 새 ID 발번. 같은 스냅샷+같은 config → 같은 결과(재현·감사 가능).
- 환율(`fx_rates`)은 **config 입력값을 스냅샷에 복사 고정**(조사 아님) → 통화 환산 재현 보장.

### 생성 순서
```
① 데이터: [EXT]·[NEWS] 조사 + [INT] 읽기 → 스냅샷 구성
② 계산  : [CALC] 연산
③ 해석  : [AI] 인사이트 (②결과 입력, 조사 아님)
④ 요약·카드: 요약 탭 + (유형2)상위3국 카드 — 맨 마지막
```
> `[AI]`는 ③ 전용. ①(조사)에 넣지 말 것 — 환각 방지.
